安装和配置 Android 开发工具。

安装 Android 开发工具

Google 为 Android 开发提供了工具。可以选择基于 Eclipse 或者 IntelliJ 之一。
这篇文章两种工具都会介绍，但重点会基于 Eclipse 工具。

选择哪个 IDE 主要是基于个人的偏好，上面两个工具 Google 团队都会全面支持的。

要求

系统要求

只要在一个合理大小的计算机上便可以完成 Android 开发。现代计算机推荐：2.6 GHz CPU，最少 8 GB 内存，一个 SSD 会显著加快 Android 模拟器的运行。

下载 Android 开发工具包

Google 提供了一个基于 Eclipse IDE 的捆绑包叫做 Android Developer Tools，这里有个链接可以找到 Google 提供的 Android 开发工具存档文件 Getting the Android SDK

独立安装的 ADT 运行

解压下载好的 zip 文件，定位到 Eclipse 文件目录，双击eclipse启动程序（如同 window 下eclipse.exe）。

更新现有的 Eclipse IDE

看 “更新现有 Eclipse IDE 支持 Android 开发” 介绍怎样更新已存在的 Eclipse IDE 来支持 Android 开发。

安装 Android SDK 版本

使用 Android SDK Manager

Android SDK Manager 允许你去安装特定的 Android API。在 Eclipse 菜单项中选择 Window → Android SDK Manager 或者运行安装目录 tools/android 程序。

你也可以从 ADT 的本地安装目录 SDK 文件目录中找到启动程序。

Android SDK Manager 允许你安装和删除 Android 开发包。

选择你想要安装的 Android SDK 版本，然后点击 Install 按钮执行安装。下面这张截图选择了 Android API 19 的平台版本：

点击 Install 按钮和接受许可协议安装所有选择的选项。安装完成后，关闭 Android SDK Manager 并重启 Eclipse IDE。

安装支持库（Support Library）

支持库可以让低版本的 Android 设备兼容和使用高版本的 Android 平台所提供的一些基础功能。

在 Android SDK Manager 中选择 Extras 并安装 Android Support Library。

Android 目前有几个版本的库，v4、v7 和 v13 版本，它们分别对应各自的 Android API 版本有效。例如，支持库v7工作于 Android API 版本号为 7 的设备。高版本的支持库也需要依赖于低版本的支持库才能正常工作。例如，支持库v7需要支持库v4。

在 Eclipse 中访问 Android SDK 源代码

安装

在开发过程中访问 Android 源代码是非常有用和必要的。

你可以通过 Android SDK Manager 下载相应版本的 Android SDK 源码包。

源代码下载后被存储在 android_sdk 安装路径的 sources/android-xx 目录下，xx 是对应的 Android API 版本号，例如，19 表示 Android 4.4 版本。

将源代码关联到 Eclipse IDE

要将 Android 平台源代码关联进 Eclipse IDE，你首先得创建一个 Android 项目，然后通过项目的android.jar文件的属性Properties添加源码包。

在创建的 Android 项目右击android.jar文件，在打开的菜单选择 Properties → Java Source Attachment。

选择 External location 和 点击 External Folder… 按钮，浏览 android_sdk 安装路径的 sources/android-xx 目录，然后点击确认按钮。

验证

验证你是否真正能查看源代码，例如，要查看Activity源代码，可以按下快捷键Ctrl+Shift+T打开对话框，输入Activity看看能不能正常打开源代码。

手动安装 Android SDK
