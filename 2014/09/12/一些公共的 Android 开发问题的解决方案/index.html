本文列出了在 Android 开发过程中常见的问题和可能的解决方案。

解决典型的 Android 开发问题

在你的 Android 开发过程中，事情并不总是做它们应该做的。本节概述了一些典型的问题以及如何解决这些问题。

清理项目

一些用户报告说，他们得到以下错误：

1、Project … is missing required source folder: ‘gen’（项目…缺少所需的源文件夹：‘gen’）

2、The project could not be built until build path errors are resolved.（构建路径错误未解决之前不能构建项目）

3、Unable to open class file R.java.（类文件 R.java 不能打开）

解决以上这些问题可以通过项目菜单选择：Project → Clean。

如果项目中引用了类项目（Library Project），确保也清理了它们。

android.content.ActivityNotFoundException

在你的应用程序启动和执行的过程中，你的应用程序被终止了。在 LogCat 你看到这样的信息：android.content.ActivityNotFoundException : Unable to find explicit activity class; have you declared this activity in your AndroidManifest.xml?

出现这种情况请检查你的 AndroidManifest.xml 文件中是否声明了相应的 activity。

Android 调试桥（adb）的问题

模拟器的通讯或你的 Android 设备可能会有问题。这种通信是由 Android 调试桥（adb）处理的。

出现这些问题 Eclipse 允许你重置 adb，切换到 DDMS 视图，选择 Window → Open Perspective → Other… → DDMS。

重启 adb，通过设备视图选择 Reset adb。

如果这样还不能工作，可以通过 adb 在命令行执行 adb kill-server，紧跟着执行 adb start-server。

LogCat

LogCat 视图显示你应用程序的日志信息和帮助你分析问题。例如，Java 程序的异常可以被显示在这里。打开这个视图，选择 Window → Show View → Other… → Android → LogCat。

Java7

如果 Android 项目提示你不能使用 Java7，你可以选中项目，右击选择 Properties → Java Compiler，确保之后使用的是 Java1.5 或 Java1.6。你可能还需要勾上 Enable project specific settings 复选框。

Eclipse 报告 R.java 文件丢失

有时候 Eclipse 会报错，例如，R.layout.main 不能被找到。检查你的源代码确保没有导入 android.R，因为 android.R 被导入后将会阻止 Eclipse 查找你的 R 文件。

解决不太典型的 Android 开发问题

这部分列出了一些有时候但不经常发生的问题。

java.lang.IllegalArgumentException: Binary XML file Duplicate id（二进制 XML 文件重复的 ID）

这种问题是由于试图加载两次相同的片段（fragment）。相同的片段在第二次被加到 FragmentManager 时抛出异常。

解决方案：确保每一个片段（fragment）仅仅只有一次被加到一个 activity 中。

Android 编辑器没有打开

Android 提供了一个非常好的编辑器来编辑 Android 资源文件，不幸的是由于错误使用 ADT，这些编辑器并不总是自动打开。如果发生这种情况,您可以手动打开这个编辑器。右键单击你的菜单文件并选择 Open with → Android Menu Editor。

模拟器没有启动

如果你的模拟器启动不起来，确保 android sdk 的路径没有包含任何空格符号。

在部署过程中超时

如果您在部署过程中面临超时问题可以增加 Eclipse 首选项的默认超时。选择 Window → Preferences → Android → DDMS 然后增加 ADB connection timeout (in ms) 的值。

安装失败由于存储不足

有时候，模拟器将拒绝安装一个应用程序带有一份错误信息：INSTALL_FAILED_INSUFFICIENT_STORAGE.

一个 Android 虚拟设备默认提供的存储 Android 应用程序的大小只有 64M 。你可以通过重启模拟器并选择 Wipe user data 标记来清理你的应用程序。

或者，你可以重新设置模拟器的分区大小，打开模拟器编辑面板，通过 New 按钮设置 Ideal size of data partition 的属性。

调试证书过期

如果你的得到一个调试证书过期（Debug Certificate expired ）的错误信息，你可以切换到 Android AVD 的文件夹下，找到 debug.keystore 文件并删除掉。因为这个文件有效期只有一年，这个文件不存在的话，Eclipse 会自动重新产生。

@Override 错误信息

@Override 注解是在 Java1.6 被引入的。如果你接收到一个 @Override 错误信息，请将 Java 编译器的级别改为 Java1.6。右击你的项目，选择 Properties → Java Compiler → Compiler compliance level，然后在下拉菜单中选择 1.6。

缺失导入

Java 要求那些不是标准 Java 语言中的类必须通过全限定名或 import 关键字导入。

如果你看到 XX cannot be resolved to a variable 的错误信息，在当前的编辑器中右键，选择 Source → Organize Imports 来导入必须的包。
